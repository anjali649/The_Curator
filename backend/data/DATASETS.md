# Problem catalog datasets

Files live in `backend/data/`. On startup, `services/dataset_service.py` loads **`problems.json`** (if present), then **merges** every file listed in `_EXTRA_PROBLEM_CSV` in order. Supplemental rows get **new numeric IDs** so they never collide with the primary catalog.

Columns for merged CSVs: `id`, `title`, `description`, `category`, `tags` (comma-separated in the file; stored as a list after load).

---

## Primary

| File | Approx. rows | Description |
|------|----------------|-------------|
| **problems.json** | ~46 | Hand-authored startup / sector problems with optional `startup_idea`. Loaded first; **takes precedence** over `problems.csv`. |

---

## Supplemental CSVs (merged automatically)

| File | Row count | What it is |
|------|-----------|------------|
| **sdg_targets.csv** | 169 | UN Sustainable Development Goal **targets** (normalized for this app). Categories use **official UN short goal titles** (e.g. *No Poverty*, *Climate Action*). To rebuild from upstream text, use the public SDG targets CSV (e.g. Data Pop Alliance mirror on GitHub) and regenerate columns. |
| **cfpb_complaints_sample.csv** | 18 | **Educational** samples styled like consumer-finance product categories—not real CFPB complaints. |
| **global_health_challenges.csv** | 15 | Curated **public health / systems** themes (synthetic, not patient data). |
| **climate_energy_challenges.csv** | 15 | Curated **climate, energy, and sustainability** themes. |
| **digital_society_challenges.csv** | 15 | Curated **cybersecurity, AI, and digital society** themes. |
| **agri_food_systems.csv** | 12 | Curated **agriculture and food systems** themes. |
| **innovation_policy_gaps_1k.csv** | **1200** | **Synthetic** cross-sector “innovation / policy gap” scenarios for demos and analytics. Generated by `scripts/generate_innovation_policy_gaps.py` (fixed seed). **More than 800 entries**, distinct from the bulk file below. |
| **large_synthetic_problems.csv** | **8000** | **Synthetic** high-volume stress-test data. Generated by `scripts/generate_large_synthetic_csv.py`. |

---

## New dataset: `innovation_policy_gaps_1k.csv` (summary)

- **Size:** 1200 problems (greater than 800 as requested).
- **Purpose:** Fill the Explorer, search, and TF–IDF + KMeans clustering with many **similarly structured** rows across **Healthcare, Education, Technology, Sustainability, FinTech, Logistics, Genomics, Cybersecurity, Agriculture, General**—each row describes a fictional **phase / stakeholder / blocker** mix (policy and innovation framing).
- **Nature:** **Synthetic** text for **education and demos only**; not real incidents, companies, or people.
- **Regenerate:** `cd backend && python scripts/generate_innovation_policy_gaps.py`

---

## Pipeline note

SQLite **`app.db`** stores **user submissions** (e.g. startup ideas), **not** these static problem tables. Clustering runs on the **merged in-memory list** at API startup.
